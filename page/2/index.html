Extends

  • Extends
    • class 자식클래스 extends 부모클래스 { } 형태.
    • 단어가 inheritance가 아닌 extends인 이유는 상속이 단순 소스를 물려받는 것만을 의미하지 않기 때문 -> 상속은 자식 클래스에서 멤버변수나 메소드를 추가로 확장했을 때 의미O.
    • 멤버 변수로 name, score를 가지며 각 major, classNumber을 멤버로 가지는 2개의 class가 있을 때 name과 score가 겹치기 때문에 상속을 적용하여 Student class 에서 name, score를 다루고 Student 클래스를 상속받는 두개의 class를 만들어 사용.
    • 논리적으로 자식 클래스가 부모 클래스에 속하는 경우 사용, 그외에는 문법적으로 가능하지만 유지/보수, 분석 쪽에서 좋지 못함
    • 단일 상속과 다중 상속도 존재하지만 자바에서는 단일 상속만 지원.
  • Extends and Overriding
    • 부모의 멤버변수를 자식 클래스에서 오버라이딩 하면 부모 변수는 상속 X
      • 부모 클래스의 멤버 변수가 name, score가 있을 때 자식 클래스에서 score, major로 멤버변수를 정의하면 부모의 변수는 상속되지 않는다.
    • 부모 클래스의 메소드를 자식 클래스에서 오버라이딩하면 부모의 메소드는 상속 X
    • 오버로딩 = 중복정의, 오버라이딩 = 재정의
      • 오버로딩은 한 클래스에서 동일한 이름의 메소드가 존재하는것, 이때 메소드 이름은 동일하지만 매개변수의 개수, 타입이 다름
      • 오버라이딩은 상속 구조에서 부모의 메소드, 멤버와 동일한 시그니처를 가진 메소드가 자식 클래스에서 재정의 되는것.
    • 부모 클래스에서 private멤버변수, 메소드는 자식 클래스로 상속되지 않는다.
    • 메소드 오버라이딩에서 부모 클래스의 접근제한보다 범위가 확대되는 것은 상관없지만 축소되는것은 안된다.
      • 부모가 public void sample 일때 자식은 private void sample로 오버라이딩 불가능
  • Extends and Constructor
    • 생성자 연속 호출
      • 부모 클래스에서 1을 출력, 자식 클래스에서 2를 출력하고 main에서 자식 객체를 생성하면 1, 2 순서로 출력. → 자식 객체생성시 부모 클래스 생성자 먼저 실행
      • 부모의 멤버 변수가 자식으로 상속 전에 반드시 초기화 되야하기 때문에 부모 생성자가 반드시 먼저 호출 되는것
    • super() constructor
      • 생성자에서 다른 생성자 호출하는 this() 처럼 부모 생성자를 super()로 호출
      • 상속관계에서 별도의 언급이 없다면 부모의 기본 생성자가 자동 호출
      • 부모 클래스에 기본 생성자가 없으면 반드시 super() 사용해 특정 생성자 명시적 호출이 필요
      • 반드시 첫 번째 줄에 작성
    • super. —> 부모 클래스의 멤버 변수나 메소드 참조에 사용 (this.과 유사)
  • final
    • 멤버변수 앞에 final을 설정하면 그 멤버변수는 상수가 된다.
    • 메소드 앞에 final을 설정하면 해당 메소드는 오버라이딩 할 수 없다.
    • class 앞에 final을 설정하면 상속을 금지한다. (public final class Name)

Type conversion of an object

  • 묵시적 변환

    • A는 하나의 멤버변수를, B는 두개의 멤버변수를 가지는데 아래 코드에서는 멤버변수를 하나 가진 A 타입의 변수에 멤버변수를 두개 가진 B타입의 변수를 저장했다.
      • 멤버변수 1개짜리에 2개짜리를 저장했기 때문에 자신보다 더 많은 변수를 가진 객체를 참조하는 것을 묵시적 타입 변환이라 한다.
      • 묵시적 타입 변환으로 가능한 문법이지만 실제 사용할 수 있는 멤버변수는 참조변수가 선언된 클래스에 속한 변수만 사용 가능 (a.i는 사용이 불가능)
    • 자식 객체는 부모 타입의 변수에 할당 가능
    • 반대로 2개 변수가진거에 1개짜리 주면 2개짜리 객체가 하나의 변수만 사용할 수 있기 때문에 데이터가 잘렸다고 표현하며 에러 발생

  • 묵시적 타입 변환 활용

    • 같은 부모 b를 상속받은 자식 c, d가 있을 때 c나 d는 서로의 타입에 할당할 수 없다.
    • 묵시적 타입 변환을 활용하면 다양한 타입의 객체를 배열 같은곳에 저장하고 사용할 수 있다. ex. A[] aList = {new A(), new B(), new C(), new D()};

  • 명시적 변환

    • 묵시적 타입 변환을 했던 객체가 원래 자신의 타입으로 되돌리는 형태의 명시적 변환만 허용한다.

Hash Table

  • Hash 란 임의 값을 고정 길이로 변환한것
  • Key에 데이터(value)를 매핑할 수 있는 데이터 구조
  • 해쉬 함수를 통해 배열에 키에 대한 데이터를 저장할 수 있는 주소(index)를 계산
  • key를 통해 바로 데이터가 저장된 주소(index)를 알 수 있어 저장 및 탐색 속도가 획기적으로 빠르다
  • 미리 해쉬 함수가 생성할 수 있는 주소에 대한 공간을 배열로 할당, 키에 따른 데이터 저장 및 탐색을 지원 (메모리 공간과 탐색 시간을 맞바꾸는 기법으로도 불림)
  • Python에서는 별도 구현 X, 딕셔너리 타입 사용

동작 방식

  1. Key(1)를 Hash fucntion(2)에 입력하면 주소(index number) 출력
  2. 출력된 주소(index number)를 (3)에서 찾아 필요한 데이터(4) 찾을 수 있다.
장점
  • 데이터 R / W 속도가 빠르다
  • 해쉬는 키에 대한 데이터가 있는지 (중복) 확인이 쉽다
단점
  • 일반적으로 저장공간이 기존보다 더 많이 필요하다
  • 여러 키에 해당하는 주소가 동일할 경우 충돌을 해결하기 위한 별도의 자료구조가 필요하다
사용
  • 검색이 많이 필요할 때
  • 저장, 삭제, 읽기가 자주 쓰일 때
  • 캐쉬 구현시 (중복 확인이 쉬워서)

Queue

  • 줄을 서는 행위같은 개념을 추상화한 데이터 구조
  • 가장 먼저 넣은 데이터를 제일 먼저 처리하는 구조 (FIFO)
  • 사용되는 메소드
    • Enqueue : 큐에 데이터 입력
    • Dequeue : 큐에 데이터 제거
  • Python 에서 Queue 기능을 제공하는 메소드를 사용하여 큐 구현 가능 (import queue)

Stack

  • 마지막에 들어온 데이터를 제일 먼저 처리하는 추상적인 데이터 구조
  • 사용되는 메소드
    • push : 스택에 데이터 입력
    • pop : 스택에 데이터 제거
  • 장점
    • 구조가 단순해서 구현이 쉽고 데이터 R/W 속도가 빠르다
  • 단점
    • 데이터 갯수를 미리 정해야 해서 메모리 낭비가 발생할 수 있다.
  • Python 에서 리스트 기능을 제공하는 메소드를 사용하여 스택을 구현할 수 있다. (append(push), pop) 메소드 제공

Linked List

  • 원소와 다음 원소의 메모리 주소를 가르키는 포인터로 이루어진 데이터 구조
  • 처음을 의미하는 head, 마지막을 의미하는 tail이 있을수 있다.
  • 장점
    • 미리 메모리 공간을 할당하지 않아도 됨
  • 단점
    • 원소 외에 포인터를 가지기 때문에 저장공간 효율이 높지 않다
    • 특정 정보를 찾을 때 속도가 느리다
    • 중간 데이터를 삭제하면 앞 뒤 연결을 재구성해야 하는 부가적 작업 필요
  • 사용자 수가 20만명으로 예상되는 웹 페이지의 회원 정보를 배열로 관리할 때 25만개의 배열을 미리 할당해야 하는데 사용자가 10만명밖에 되지 않거나 25만명을 초과할 때 문제가 생길 수 있다, 하지만 Linked List를 사용하면 회원 한명이 가입할 때마다 노드를 하나씩 생성하므로 메모리를 정확하게 사용할 수 있다.

인스턴스 생성과 힙 메모리

인스턴스 (instance)
  • 클래스는 객체의 속성을 정의, 기능을 구현하여 만들어 놓은 코드 상태
  • 실제 클래스 기반으로 생성된 객체(인스턴스)는 각각 다른 멤버 변수 값 가짐
  • new 키워드를 사용하여 인스턴스 생성
힙 메모리 (heap memory)
  • 생성된 인스턴스는 동적 메모리 (heap memory) 에 할당됨
  • C나 C++ 언어에서는 사용한 동적 메모리를 프로그래머가 해제 시켜야 하지만 자바에서는 Gabage Collector가 주기적으로 사용하지 않는 메모리 해제
  • 하나의 클래스로 부터 여러개의 인스턴스가 생성되고 각각 다른 메모리 주소를 가지게 됨

생성자 (constructor)

생성자
  • 오직 객체를 생성하기 위해 new와 함께 호출 className instanceName = new className(); 에서 className()이 생성자
  • 객체가 생성될 때 변수나 상수를 초기화 하거나 다른 초기화 기능을 수행하는 메소드 호출
  • 반환값이 없고 클래스 이름과 동일
  • 대부분 생성자는 외부 접근이 가능하지만 필요하다면 private으로 선언할수도 있다.
  • this 는 변수나 메소드 앞에 사용되며 this 사용시 매개변수가 아닌 멤버 변수를 의미
  • 맴버변수들의 값을 모두 default로 초기화
기본 생성자 (default constructor)
  • 클래스에 반드시 하나이상 존재
  • 생성자를 구현하지 않아도 new 키워드와 함께 생성자를 호출 가능
  • 클래스 생성자 하나도 없는 경우 컴파일러가 생성자 코드 넣어준다 (public className(){})
  • 매개변수, 구현부가 없다
접근 제어자
  • Public
    • 다른 패키지에서 사용 가능
    • 다른 어떤 클래스에서도 public class는 사용 가능
  • Private
    • private이 붙은 변수나 메소드는 해당 클래스에서만 접근이 가능
    • 다른 클래스에서 접근이 불가능
  • Default
    • 접근 제어자를 별도로 설정하지 않는다면 해당 패키지 내에서만 접근이 가능
    • **동일 패키지**, 다른 클래스에서 접근 가능
  • Protected
    • 이 제어자가 붙은 변수, 메소드는 동일 패키지의 클래스 또는 해당 클래스를 상속받은 다른 패키지의 클래스에서만 접근이 가능 (상속 → import packageName.ClassName;)
패키지 (package)
  • 소문자로만 구성
  • 여러 디렉토리로 구성되면 . 을 이용해 구분 (package school. high)
  • 주석을 제외하고 클래스 파일의 가장 첫 줄에 위치
  • import packageName.ClassName; 다른 패키지의 클래스 사용하려면 있어야됨

함수와 메소드

함수

  • 하나의 기능을 수행하는 일련의 코드
  • 구현(정의)된 함수는 호출하여 사용, 호출 함수는 메모리와 제어를 할당받고 기능이 끝나면 제어와 메모리 반환
  • flow : main이 작동할때 스택에 main의 메모리 할당, 메소드 호출시 스택에서 main의 위에 메소드를 위한 메모리 할당, 메소드 종료 시 스택에서 메소드의 메모리 삭제

메소드

  • 객체의 기능 구현을 위한 클래스 내부에 구현되는 함수
  • 메소드를 구현함으로 객체의 기능 구현
  • 메소드의 이름은 그 객체를 사용하는 client에 맞게 작성

Class

  • student class (객체의 틀)를 생성하고 studentTest class에서 사용하려면 Student name = new Student(); 같이 인스턴스를 생성해야됨
  • name.methodName(); 같은 방법으로 student class의 메소드도 사용 가능
  • 클래스 하나를 기반으로 여러 인스턴스 생성 가능 (바로 위를 예시로 들면 학생은 한명이 아니라 여러명이기 때문에)
  • student class는 객체의 틀만 제공했기 때문에 인스턴스 생성 후 멤버 변수는 따로 입력

Array

JAVA 특징

  • C, C++과 달리 pointer를 사용하지 않기 때문에 메모리를 직접 지정하지 않아서 프로그램이 더 안정적
  • 개발에 필요한 JDK가 있고, 실행을 하기 위한 JRE가 있다.
  • JAVA는 다형성 = 상속 + 메소드 오버라이딩 + 형변환 이라는 프로그래밍 언어 고유의 특징이 있다.
  • 컴파일러가 하나가 아닌 두개(1, 2가 컴파일러)
    • JAVA -> (소스코드.java) -> 컴파일러(1) -> (바이트 코드.class) 여기까지 플랫폼(OS) 영향이 없다.
    • JVM (2) 에서 각 플랫폼 (OS) 별로 똑같이 실행 가능

JAVA 자료형

  • 메모리 크기가 정해진 기본 자료형 (int, char, double, boolean etc.)
  • 클래스에 따라 사용 크기가 달라지며 JDK에서 제공되는 클래스와 프로그래머가 정의하는 참조형 자료형이 있다.

형 변환

  • 서로 다른 자료형 값이 대입될 때 형 변환이 일어난다.
  • 묵시적 형 변환 : 큰 수나 더 정밀한 수로 대입될때
  • 명시적 형 변환 : 변환될 자료 형을 명시하는 것으로 자료 손실이 발생할 수도 있다.

단란 회로 평가

  • 논리곱 (&&) 은 두 항이 모두 true 일때 결과가 true. 앞의 항이 false면 뒤 항 결과 평가X false (& 하나만 사용하면 앞, 뒤항 모두 평가)
  • 논리합 (||) 은 두 항 모두 false 일때 결과가 false, 앞 항이 true면 뒤 항 결과 X true (| 하나만 사용하면 앞, 뒤항 모두 평가)

비트 연산자

  • 마스크 : 특정 비트를 제외하고 몇 개의 비트 값만 사용할 때 사용
  • 특정 비트를 1로 설정하여 사용 가능 ( & 00001111 하면 하위 4비트만 사용 가능 )
  • | 11110000 으로 원하는 비트 원하는 자리의 비트만 사용가능

Git revert

  • Rename
    • 파일의 이름을 바꿀때 사용
    • a.md를 b.md로 mv명령어로 이름을 바꾼다면 git에서는 a.md가 삭제되고 b.md를 untracked로 인식
    • git mv 명령어를 사용하면 git에서도 제대로 인식
  • Undoing (commit 하지 않은 작업 취소)
    • 자주 사용하는 명령어로 변경사항을 취소하기 위한 명령
    • git restore name -> name의 변경사항이 취소 (원상복구)
    • git restore . -> 현재 디렉토리 아래의 모든 파일 복구
  • Unstaging (add로 staging 한 작업 되돌림)
    • 명령에 reset이 들어가있어 팀 전체에 영향을 줄 수 있어 왠만하면 사용 자제
    • git reset HEAD name : add한 작업 취소
  • Edit latest commit (commit massage 수정)
    • git commit –amend (직전 commit 다시 열어서 수정)
    • rebase 명령으로 직전 말고도 commit 수정가능
    • commit을 대충 할수도 있기 때문에 사용을 자제
  • Revert (commit을 되돌릴 때 사용)
    • git revert –no-commit HEAD~n.. -> n개의 commit을 되돌리고 되돌리는 commit massage 작성
    • –no-commit 안붙이면 n개 취소할 때 n개의 commit 작성해야됨

Git Team practice

  1. 팀장이 repo 생성, git clone 후 git flow init
  2. 팀원이 팀장의 repo를 fork, fork한 URL로 git clone
  3. 팀원은 issue를 작성하고 git flow init 으로 develop으로 가면 팀장과 같은 상태가됨
  4. git flow feature start name 으로 기능별로 작업하고 add, commit
  5. git flow feature finish name 으로 feature 브렌치 작업 종료하고 git push -u origin develop
  6. fork한 페이지로 가서 pull request
  7. 팀장이 작업 내용을 확인해보고 merge
  8. git remote add upstream 팀장repo URL 으로 upstream 주소 추가
  9. git pull upstream develop
  10. 어느정도 작업내용이 쌓이면 팀장이 git flow release로 테스트하고 정상적이라면 main으로 release
