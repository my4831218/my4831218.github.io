Database

  • SELECT * FROM TABLE 테이블 확인

DML (Data Manipulation Language)

  • INSERT
    • 데이터베이스에 데이터 입력
    • INSERT INTO tableName(columnList) VALUES(valueList);
    • ex. INSERT INTO STUDENT VALUES(’STD-002’, ‘이름’, ‘54’, ‘전공’);
  • UPDATA
    • 데이터베이스에 저장된 데이터 수정
    • UPDATE TableName SET columnName = 'values' (, 로 한번에 여러개 수정 가능)
  • DELETE
    • 데이터 베이스에 데이터 삭제
    • DELETE TableName WHERE columnName = 'value'

DQL(Data Query Language)

  • 처리 순서 select - from - where - group by - having - order by

  • SELECT (출력될 테이블 목록들이라 생각)

    • WHERE
      • 비교 : =, <>, <, <=, >, ≥
      • 범위 : BETWEEN
      • 집합 : IN, NOT IN
      • 패턴 : LIKE
      • NULL : IS NULL, IS NOT NULL
      • 복합조건 : AND, OR, NOT
    • ORDER BY
      • 정렬, 숫자, 날짜, 문자, NULL 등등으로 정렬
      • SELECT * FROM table ORDER BY column1 DESC, column2 AS; column1 은 내림차순, 2는 오름차순 정렬
    • 산술식 검색
      • select 컬럼1, 컬럼2/1000 from table 컬럼2를 1000으로 나눈 테이블 보여줌
    • AS
      • SELECT 콜럼, SUN AS 임시컬럼이름 from 임시 컬럼이름으로 결과 테이블 출력
    • GROUB BY
      • select 컬럼 from 테이블 group by 컬럼 having count (*) ≥ 3; 컬럼에 3개이상 있는 목록 출력

  • JOIN

    • INNER JOIN

      • 교집합 개념 기존 테이블과 join 테이블의 중복값 보여줌
      • 기준 테이블, 조인 테이블의 조인 컬럼의 값이 모두 존재하는 경우에만 데이터 조회
      • SELECT 컬럼이름 FROM 기준테이블 INNER JOIN 조인테이블 ON 기준테이블
      • ex. SELECT A.NAME, A.HEIGHT, A.WEIGHT, B.성별 FROM A INNER JOIN B ON A.WEIGHT = B.WEIGHT 명령하면 다음과 같이 출력, test2 는 NULL이 있어 출력 X

    • OUTER JOIN

      • LEFT, RIGHT 등은 기준 테이블을 지정, join 테이블의 데이터가 없어도 기준 테이블의 모든 데이터 조회, 조인 테이블에 데이터가 존재하면 해당 데이터 참조 가능
      • ex. SELECT A.NAME, A.HEIGHT, A.WEIGHT, B.성별 FROM A LEFT JOIN B ON A.WEIGHT = B.WEIGHT
