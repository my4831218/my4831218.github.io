Java-day8

Posted on 2022-04-15 In Java

instanceof

  • 원래 인스턴스의 형이 맞는지 여부 체크, boolean타입으로 반환
  • if (A instanceof B)가 주로 사용, 상속받은 조상과 비교해도 True 반환

Template method

  • 추상 메소드나 구현된 메소드를 활용해 코드의 흐름을 정의하는 메소드
  • final로 선언해 하위 클래스에서 재정의 불가능하게 만듦
  • 프레임워크에 많이 샤용되는 설계 패턴
  • 추상 클래스로 선언된 상위 클래스에서 템플릿 메소드를 활용해 전체 흐름을 정의하고 하위 클래스에서 다르게 구현되야 하는 부분 추상 메소드로 선언해 하위 클래스에서 구현

tip

  • Define class를 만들어 public static final로 자주쓰는 상수들을 만들어두면 Define. 으로 접근하여 쉽게 사용이 가능

Interface 예시

  • 여러가지 DB 제품이 지원될 수 있는 객체 구현
  • 이미지의 if (인터페이스) 에 제공되야할 메소드를 선언
  • 1, 2, 3에 각각 implements 하고 메소드 오버라이딩으로 기능 구현

Class

  • Java에서 모든 클래스와 인터페이스는 컴파일 후 class파일이 생성
  • Class 클래스는 컴파일된 class파일 로드, 객체를 동적으로 로드하고 정보를 가져오는 메서드 제공
  • Class.forName(”클래스 이름”) 메소드로 클래스를 동적으로 로드, 클래스 이름으로 직접 가져오거나 생성된 인스턴스에서 가져올 수 있다.
    • Class test = Class.forName(”java.lang.String”);
    • Class test = String.class();
    • String test = new String(), ClassName test1 = s.getClassName(); (-> )
      getClass로 가져오려면 인스턴스가 만들어진 상태여야 한다.

동적 로딩

  • 컴파일 시 데이터 타입이 binding 되는 것이 아닌 실행중에 데이터 타입을 binding 하는 방법
  • 프로그래밍 시 문자열 변수로 처리, 런타임 시 원하는 클래스를 로딩하여 binding하는 장점이 있다
  • 컴파일 시 타입이 정해지지 않아 동적 로딩시 오류가 발생하면 프로그램에 심각한 장애가 발생할수도 있다.
