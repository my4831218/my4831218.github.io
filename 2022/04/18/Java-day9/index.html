Generic

Inner Class

  • 클래스 내부에 선언한 클래스, 외부 클래스와 밀접한 연관이 있는 경우가 많고 다른 클래스에서 사용할 일이 거의 없는 경우 선언하여 사용

Instance inner class

  • 내부적으로 사용할 클래스 선언으로 private선언 권장
  • 외부 클래스 생성 후에 생성됨

    Static inner class

  • 외부 클래스 생성과 무관하게 사용
  • 일반 메소드는 외부 클래스의 변수를 제외하고 모든 변수 사용, static 메소드는 외부, 내부의 static 변수만 사용 가능

Local inner class

  • 지역 변수와 같이 메소드 내부에서 정의해서 사용
  • 메서드 호출이 끝나면 메소드에 사용된 지역변수의 유효성이 사라짐
  • 메소드 호출 이후에도 사용해야 할 경우도 있을 수 있어 지역 내부 클래스에서 사용하는 메소드의 지역 변수나 매개 변수는 final로 선언

Anonymous inner class

  • 클래스 이름을 생략하고 주로 인터페이스나 추상 클래스를 구현하여 반환
  • 인터페이스나 추상 클래스 자료형의 변수에 직접 대입해 클래스 생성 or 내부 클래스의 메소드 내부에서 생성해 반환
  • widget의 이벤트 핸들러에 활용

람다식

  • 함수형 인터페이스를 사용
  • 익명 함수와 매개 변수만으로 구현되므로 인터페이스에서 단 하나의 메소드만 선언
  • 함수형 인터페이스는 내부에 여러개 메소드 선언시 에러

Stream

  • 자료의 대상과 관계없이 동일한 연산 수행
  • 한번 생성하고 사용한 스트림은 재사용 X
  • 기존 자료를 변경하지 않는다
  • 중간 연산과 최종 연산으로 구분
  • 중간연산
    • filter() 조건에 맞는 원소 추출, map() 조건에 맞는 원소 변환, sorted()
  • 최종연산
    • forEach() 원소를 하나씩 꺼냄, count() 원소 개수. sum() 원소들의 합
  • 중간연산에서 스트림에 자료가 들어가고 최종 연산하면서 스트림의 자료를 소모하는 구조인것 같다
  • ex. Arrays.stream(배열이름).method(), 배열이름.stream().중간연산().최종연산()
