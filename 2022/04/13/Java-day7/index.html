Java-day7

Posted on 2022-04-13 In JAVA

Static Method

desc

  • 외부에서 static 변수를 참조할 수 없기 때문에 method를 만들어 접근
  • static method는 인스턴스 생성과 무관하게 클래스 이름으로 호출 가능
  • 인스턴스 생성 전에 호출될 수 있기 때문에 인스턴스 변수 사용 불가능
  • 프로그램이 메모리에 있는 동안 계속 존재하기 때문에 큰 메모리 할당은 좋지 않다

Singleton pattern

  • 프로그램에서 인스턴스가 단 한 개만 생성되야 하는 경우 사용하는 디자인 패턴
  • static 변수, method를 활용하여 구현
  • 외부 접근이 불가능한 private static로 instance 생성, 외부에서 사용을 위한 public static getInstance 메소드

Singleton pattern 실습

Memory

변수 유형 선언 위치 사용 범위 생성메모리 생성과 소멸
지역 변수(local) 함수 내부 함수 내부에서만 스택 함수 호출될때 생성, 함수 종료 시 소멸
멤버 변수(instance) 클래스 멤버 변수 클래스 내부, private이 아닐 때 참조변수로 다른 클래스에서 사용 힙 인스턴스 생성시 힙에 생성, 가비지 콜렉터가 메모리 수거할 때 소멸
static 변수 static 사용 클래스 내부 클래스 내부, private이 아닐 때 클래스이름으로 다른 클래스에서 사용 데이터 프로그램 처음 시작할 때 상수와 함께 데이터 영역에 생성, 프로그램 끝나고 메모리 해제할 때 소멸

Array

Object Array

  • 객체의 주소가 들어갈 메모리가 할당됨, 각 객체는 따로 생성, 저장
  • System.arrayCopy(복사할 array, 복사array 시작점, 붙여넣을 array, 붙여넣을 array의 위치, 복사할 길이) 자바에서 제공되는 배열 복사 메소드로 복사 가능 (얕은복사)
    • 얕은 복사 : 객제 추소만 복사되어 한쪽 배열 요소를 수정하면 같이 수정됨
  • 깊은 복사 : 각각의 객체를 생성해서 그 객체의 값을 복사하여 배열이 서로 다른 객체를 가르킴

ArrayList

  • 기존 배열은 배열의 길이를 정하고 요소의 개수가 배열보다 커지면 배열을 재할당하고 복사해야함
  • 배열의 요소를 추가, 삭제하면 다른 요소의 이동 구현
  • 객체 배열을 좀더 효율적 관리를 위해 자바에서 제공하는 클래스
  • 많은 메소드가 최적 알고리즘으로 구현되어 각 메소드의 사용 방법만 익히면 됨

ArrayList method

  • boolean add(E e) : 배열에 요소를 추가 (E는 자료형)
  • int size() : 배열에 추가된 전체 요소 개수 반환
  • E get(int index) : index 위치에 있는 요소 값 반환
  • E remove(int index) : index위치에 요소 값 제거, 그 값을 반환
  • boolean isEmpty() : 빈 배열인지 확인

ArrayList 실습

Exception 2

throws

  • throws를 이용하여 특정 메소드에서 발생된 예외를 메소드를 호출한 쪽으로 던질 수 있다.
  • 메소드마다 throws를 작성하면 한곳으로 예외를 몰수 있다. (예를들면 main으로 몰아 예외를 일괄처리)
  • , 를 사용해서 한번에 여러개의 예외를 던질 수 있다
  • Exception을 throws 하면 예상치 못한 예외까지 포함해서 한번에 예외를 처리한다.

throw 예약어

  • throw 예약어를 사용하여 강제적으로 예외를 발생시켜 프로그램을 중단시킬수 있다.
  • ex. If (test < 0) { throw new IllegalArgumentException(); }

상속된 exception method

  • 예외 객체를 생성할 때 인자를 입력하면 사용할 수 있다.

사용자 지정 예외

java.lang

  • java 프로그램에 사용하는 필수 클래스를 모아놓은 패키지
  • import로 선언하지 않아도 사용 가능
  • 최상위 클래스 Object 등 다양한 클래스 제공

Object class

  • class를 선언하면 자동으로 extends java.lang.Objecct가 추가되어 Object클래스를 상속한다.
  • 왜 상속하는가
    1. java class로서 반드시 가져야할 공통 메소드를 object 클래스로부터 상속받기 때문에 상속
    2. 모든 java 객체를 하나의 타입으로 묶어 다형성을 적용하기 위해 상속

hashCode() Method

  • Object 클래스가 제공하는 메소드
  • 생성된 객체의 메모리 주소정보를 반환
  • 똑같은 멤버변수값을 가진 객체 a, b가 있을 때 a의 해시코드와 b의 해시코드 값은 다르다.
  • 하지만 두 객체를 같은 객체로 처리할 일이 생긴다면 멤버변수를 기준으로 해시 값을 생성해야 하기 때문에 hashCode() 메소드가 상속되지 않도록 오버라이딩 해야한다.
  • 그렇게 처리하면 a의 해시코드와 b의 해시코드 값이 같아진다.

euqals() Method

  • 객체의 주소가 아닌 내용을 비교하기 위한 메소드
  • equals 메소드는 객체의 주소를 비교하는 == 연산자와 동일하게 동작한다
  • 따라서 equlas 메소드가 객체의 내용을 비교할 수 있도록 오버라이딩 해야한다.

toString() Method

  • 객체가 소유한 멤버변수에 할당된 값을 하나의 문자열로 구성하여 반환
  • default는 클래스의 이름과 객체의 메모리 주소를 결합한 문자열 반환
  • 실제 객체가 가진 멤버변수의 값을 기반으로 객체 정보를 반환하도록 오버라이딩 해야됨

String class

  • charAt(int idx) : char idx에 위치한 문자 리턴
  • length() : int 문자열 길이 반환
  • startsWith(Strng prefix) : Boolean 특정 문자로 시작하는지 여부 반환
  • substring(int beginIdx) : String 특정 인덱스 이후 문자열 반환
  • substring(int start, int end) : String start 인덱스부터 다른 end 인덱스까지 문자열 반환
  • split(String regex) : String[] 특정 문자열 구분자로 문자열을 분해하여 문자열 배열로 반환

StringBuffer class

  • 고정길이인 String 객체는 한번 생성되면 변경되는 것이 아닌 새로운 문자열 객체를 만든다
  • String a = “1” + “2” + “3” + “4”; —> 문자열 1개가 변형되는것이 아니라 문자열 5개가 생성되는 것임.
  • StringBuffer는 가변길이 문자열을 제공, StringBuffer a = new StringBuffer(”test”); —> a = a.append(”add”); 하면 문자열 1개가 생성되고 그것이 변경되는것

Wrapper class

  • 기본타입의 데이터를 객체로 감싼다는 의미로 해석 —> 기본타입을 감싸 클래스 형태로 포장

java.util

java.util.Random

  • 랜덤한 숫자 생성에 사용, nextint(n) 메소드는 랜덤한 정수 발생시킬 때 사용, 0~n-1 사의 랜덤한 정수 생성

java.util.Arrays

  • 배열에 저장된 데이터 정렬, 특정 배열 내용을 복사해 새로운 배열 객체 만들기에 사용
  • sort(array) 정렬 메소드, copyOf(array, 복사할 원소수)

java.util.Data, java.util.Calendar

  • 날짜 데이터, 정교한 날짜와 시간 정보가 필요할 때 사용
  • Date date = new Date(); —> data.toString(); Fri Oct 08 17:38:53 KST 2021 형태로 출력
  • Calendar currentDate = Calendar.getInstance(); 매수 상세하게 출력(지역, 도시, 초단위 등등)

Collection

표시된곳은 interface, 나머지는 class

ArrayList

  • 데이터 저장에 add() 메소드 사용, 데이터 꺼낼 때 get() 메소드 사용
  • add() 매개변수 타입은 Object —> ArrayList에 데이터 저장은 무조건 Object로 변환되어 저장 —> ArrayList에는 자바의 모든 타입의 데이터 저장 가능 (묵시적 형변환)
  • get() 메소드 인자는 인덱스 번호, 데이터 꺼낼 때 원래의 타입으로 변환해야 한다. (명시적 형변환)
  • 모든 종류의 데이터 저장이 가능하지만 실제로 프로그램에서 타입이 다른 데이터를 컬렉션에 저장하는 경우는 거의 없다.
  • generic으로 실수로라도 다른 타입 데이터 저장하지 않도록 안전장치 같은 느낌
  • ex. List name = new ArrayList();

    ArrayList method

  • 기본 생성자로 객체를 생성하면 10개의 데이터를 저장할 수 있고 추가되는 데이터에 따라 자동으로 공간이 10개씩 증가한다.
  • 원소를 삭제할 때 0~4까지 있을 때 1을 삭제하면 1 뒤를 쭉 앞으로 이동시켜 0,2,3,4가 된다.
  • size() : int 등록된 객체 개수 반환
  • add(Object obj) : boolean : 특정 객체 등록, 성공하면 t 실패하면 f 반환
  • get(int index) : Object index 위치에 있는 객체 반환
  • remove(int index) : Object index 위치 객체 삭제, 삭제된 객체 반환
  • remove(Object obj) : boolean 특정 객체 삭제, 성공 t 실패 f 반환
  • contains(Object obj) : boolean 특정 객체 존재 확인, 있으면 t, 없으면 f

    Set

  • List와의 차이점은 데이터 순서가 정해져 있지 않다는 것과 데이터 중복을 허용하지 않는것
  • 하지만 같은 값을 가진 객체가 중복 저장이 된다 why? —> 새로운 객체가 등록될 때 자신이 가진 객체들 중 같은 값을 가진 객체가 존재하는지 확인하는데 이때 equlas()와 hashCode() 메소드를 사용하기 때문에 두 메소드를 오버라이딩 해야한다.

    HashMap

  • 검색이 빠르다
  • 저장에 put(object key, object value) 메소드, 검색에 get(object key) 메소드 사용
  • HashMap에 사용하는 Key 데이터는 중복되지 않는 유일한 문자열이어야 한다
    • 중복되면 value에 해당하는 객체를 덮어쓰기 한다.
  • key 목록이나 Value 목록만 별도로 추출하여 사용 가능
  • keySet(); 메소드는 키 목록만 추출, values(); 메소드는 벨류목록만 추출
  • HashTable과 개념과 사용방법이 동일
