HashTable 2

Collision algorithm

  • HashTable에는 충돌(중복)이 발생하는것이 가장 큰 문제이다, 충돌을 처리하기 위한 알고리즘이 몇가지 있다.
  • Chaining
    • 개방 해슁 또는 Open Hashing 기법으로 해쉬 테이블 저장공간 외의 공간을 활용
    • 충돌이 일어나면 연결리스트 자료구조를 사용해 데이터를 추가로 뒤에 연결시켜 저장
    • 키, 벨류로 저장되있던 데이터를 키+벨류로 이뤄진 연결리스트로 구현 후 뒤에 연결시켜 데이터를 저장
  • Linear Probing
    • 폐쇄 해슁 또는 Close Hashing 기법으로 해쉬 테이블 저장공간 안에서 충돌 문제 해결
    • 충돌이 일어나면 해당 hash address의 다음 address부터 맨 처음 나오는 빈공간에 저장
      - 저장공간 활용도 높이는 기법
    • 키가 중복인지 확인하기 위해 key, value를 같이 저장한다.
  • 그 외에 충돌을 방지하기 위한 기법으로는 해쉬 함수를 재정의 하고 해쉬 테이블 공간을 확대한다 (통상 2배로)
  • 시간 복잡도
    • 충돌없는 일반적인 경우 O(1)
    • 충돌이 모두 발생하는 최악의 경우 O(n)
      - 해쉬 테이블은 일반적인 경우를 기대하고 만들어 시간복잡도 O(1)
      - ex. 16개 배열에 데이터 저장, 검색 O(n)
      - ex. 위 해쉬 테이블에 데이터 저장, 검색 O(1)

Tree

  • Node, Branch를 이용해 사이클을 이루지 않도록 구성한 데이터

    Binary Search Tree

  • 이진 트리는 노드의 자식이 최대 2개인 트리
  • 이진 탐색 트리는 이진 트리에 왼쪽 노드는 해당 노드보다 작은 값, 오른쪽 노드는 해당 노드보다 큰 값을 가진다는 조건이 있다.
  • 주 용도는 데이터 검색이며 탐색 속도를 개선할 수 있는 장점이 있다.
  • 탐색에 걸리는 시간은 O(depth)
  • n개의 노드를 가진다면 h = log n —> O(log n) (실행 한번에 탐색군이 반씩 감소)
  • 평균 시간 복잡도는 O(log n) 이지만 트리가 균형잡힌 평균 시간복잡도이고 한쪽으로 치우친 (오른쪽 자식만 가진) 이진탐색 트리라면 연결리스트와 동일한 성능 O(n)임

Heap

  • 데이터에서 최대값과 최소값을 빠르게 찾기 위해 고안된 완전 이진 트리
    • 삽입할 때 최하단 왼쪽 노드부터 순차적 삽입
  • 최대값을 구하기 위한 최대힙과 최소값을 구하기위한 최소힙으로 분류
  • 각 노드의 값이 해당 노드의 자식 노드가 가진 값보다 크거나 같아야 한다 (최대힙 기준)
    • 최소힙은 반대로 작거나 같아야 한다.
  • n개의 노드를 가진 heap 에서 데이터 삭제/삽입 시 최악의 경우 leaf 노드까지 비교하므로 시간복잡도는 O(log n)

Heap 사용 이유

  • 배열에 데이터를 넣고, 최대값과 최소값을 찾으려면 O(n) 이 걸림
  • 이에 반해, 힙에 데이터를 넣고, 최대값과 최소값을 찾으면,  O(log n)이 걸림
  • 우선순위 큐와 같이 최대값 또는 최소값을 빠르게 찾아야 하는 자료구조 및 알고리즘 구현 등에 활용

이진 탐색 트리와 비교

  • 이진 트리라는 공통점
  • 힙은 왼쪽 자신이 작은값, 오른쪽 자식이 높은값이라는 조건 없음
  • 탐색보다는 최대, 최소값 검색을 위한 구조
  • 이진탐색트리에서 왼자식 < 본인 < 오른자식 이지만 힙에서는 부모 > 자식.

구현

  • 힙은 일반적으로 배열로 구현, 배열의 시작은 0이지만 root의 인덱스를 1로 지정하면 구현이 쉬워진다.
    • 부모 인덱스 = 자식 인덱스 // 2
    • 왼쪽 자식 인덱스 = 부모 인덱스 * 2
    • 오른쪽 자식 인덱스 = 부모 인덱스 * 2 + 1

Static Method

desc

  • 외부에서 static 변수를 참조할 수 없기 때문에 method를 만들어 접근
  • static method는 인스턴스 생성과 무관하게 클래스 이름으로 호출 가능
  • 인스턴스 생성 전에 호출될 수 있기 때문에 인스턴스 변수 사용 불가능
  • 프로그램이 메모리에 있는 동안 계속 존재하기 때문에 큰 메모리 할당은 좋지 않다

Singleton pattern

  • 프로그램에서 인스턴스가 단 한 개만 생성되야 하는 경우 사용하는 디자인 패턴
  • static 변수, method를 활용하여 구현
  • 외부 접근이 불가능한 private static로 instance 생성, 외부에서 사용을 위한 public static getInstance 메소드

Array

Object Array

  • 객체의 주소가 들어갈 메모리가 할당됨, 각 객체는 따로 생성, 저장
  • System.arrayCopy(복사할 array, 복사array 시작점, 붙여넣을 array, 붙여넣을 array의 위치, 복사할 길이) 자바에서 제공되는 배열 복사 메소드로 복사 가능 (얕은복사)
    • 얕은 복사 : 객제 추소만 복사되어 한쪽 배열 요소를 수정하면 같이 수정됨
  • 깊은 복사 : 각각의 객체를 생성해서 그 객체의 값을 복사하여 배열이 서로 다른 객체를 가르킴

ArrayList

  • 기존 배열은 배열의 길이를 정하고 요소의 개수가 배열보다 커지면 배열을 재할당하고 복사해야함
  • 배열의 요소를 추가, 삭제하면 다른 요소의 이동 구현
  • 객체 배열을 좀더 효율적 관리를 위해 자바에서 제공하는 클래스
  • 많은 메소드가 최적 알고리즘으로 구현되어 각 메소드의 사용 방법만 익히면 됨

ArrayList method

  • boolean add(E e) : 배열에 요소를 추가 (E는 자료형)
  • int size() : 배열에 추가된 전체 요소 개수 반환
  • E get(int index) : index 위치에 있는 요소 값 반환
  • E remove(int index) : index위치에 요소 값 제거, 그 값을 반환
  • boolean isEmpty() : 빈 배열인지 확인

throws

  • throws를 이용하여 특정 메소드에서 발생된 예외를 메소드를 호출한 쪽으로 던질 수 있다.

throw 예약어

  • throw 예약어를 사용하여 강제적으로 예외를 발생시켜 프로그램을 중단시킬수 있다.
  • ex. If (test < 0) { throw new IllegalArgumentException(); }

상속된 exception method

  • 예외 객체를 생성할 때 인자를 입력하면 사용할 수 있다.

hashCode() Method

  • Object 클래스가 제공하는 메소드

  • 하지만 두 객체를 같은 객체로 처리할 일이 생긴다면 멤버변수를 기준으로 해시 값을 생성해야 하기 때문에 hashCode() 메소드가 상속되지 않도록 오버라이딩 해야한다.
  • 그렇게 처리하면 a의 해시코드와 b의 해시코드 값이 같아진다.

euqals() Method

  • 객체의 주소가 아닌 내용을 비교하기 위한 메소드

  • 따라서 equlas 메소드가 객체의 내용을 비교할 수 있도록 오버라이딩 해야한다.

  • 실제 객체가 가진 멤버변수의 값을 기반으로 객체 정보를 반환하도록 오버라이딩 해야됨

  • StringBuffer는 가변길이 문자열을 제공, StringBuffer a = new StringBuffer(”test”); —> a = a.append(”add”); 하면 문자열 1개가 생성되고 그것이 변경되는것

Abstract

  • 상속 구조에서 최상위 부모 클래스로만 사용
  • 추상 메소드는 시그니처만 있고 구체적 로직이 없다. ({} ←이게 없다.)
  • 예를들어 멤버변수와 메소드가 똑같은 SamsungTV 클래스와 LGTV클래스가 존재하고 main에서 SamsungTV클래스를 사용하다가 LGTV 클래스를 사용하려면 많은 코드를 사용해야 한다.
  • 이 때 다형성을 적용(상속, 메소드 오버라이딩, 타입 변환 필요)하면 소스(코드) 수정을 최소화하면서 프로그램 수정이 가능
    1. SamsungTV, LGTV와 멤버변수가 동일, 추상 메소드를 선언한 추상 클래스 TV 생성
    2. TV클래스를 SamsungTV, LGTV클래스에 각각 상속하고, 메소드 오버라이딩
    3. main 에서 사용할 때 삼성tv라면 TV 거실 = new SamsungTV();
    4. 거실에 Samsung TV를 사용하다가 LGTV로 바꿧을 때 참조변수의 SamsungTV 객체를 LGTV로 수정하면 다른부분 수정이 필요없이 기존 사용하던 켜기, 끄기, 켜기예약 모든 메소드를 사용 가능하다.

Interface

Create

  • interface name {}으로 작성하며 상수, 추상 메소드만 가지는 특수 클래스
  • 상수는 멤버변수와 구분하기 위해 모두 대문자로 작성 및 단어와 단어를 _ 로 연결
  • interface에서 상수 앞에는 static final이 컴파일하며 자동으로 추가된다.
  • interface에서 추상 메소드 앞에 abstract를 생략해도 컴파일 시점에 자동 추가

Implements

  • 추상 클래스처럼 상속 구조상 최상위 부모 클래스로만 사용, interface는 상속할 때 extends가 아닌 implements(구현하다) 사용 —> class Name implements Interface {}
  • 한 클래스에 여러개의 interface를 상속할 수 있다. —> 인터페이스가 상수만 허용하기 때문에 가능 (두 인터페이스에서 중복되는 상수라도 static이 붙어있기 때문에 이름과 결합하여 사용해서 문제가 없다) [class Name implements Interface1, Interface2 {}]
  • Interface는 부모 Interface를 상속받을 수 있다. —> 이때는 extends를 사용한다. (다중도 가능)
  • Abstract와 Interface의 다른점은 Abstract는 논리적으로 같다, 포함한다는 관계인데 그러한 관계가 아닌 논리적으로 좀 다른 관계를 하나로 묶어서 사용하기 위해서는 최상위 부모로 논리적으로 같지 않아도 객체들을 모두 포함할 수 있는 Interface를 사용해야 한다.

Exception

Exception

  • 예외는 발생할 수도있고 안할 수도 있다 (2개의 정수를 입력받는 프로그램에 문자를 입력하거나 입력하지 않는 행위 등등이 모두 예외)
  • JVM에서는 예외가 발생되면 해당 예외 클래스의 객체를 생성하여 실행중인 프로그램에 던진다(throw), 이 때 던져진 예외객체를 잡아서(catch) 처리해야 하는데 예외를 잡지 않으면 프로그램이 비정상적으로 종료.
  • 예외 객체를 처리하는 가장 기본적인 방법은 try ~ catch문 사용

Multiple catch

  • try 블록에서 2가지 예외를 예상하고 처리했는데 예상치 못한 다른 예외가 발생했을 때 두 가지 방법으로 처리가 가능하다.
    1. 새롭게 발견된 예외를 처리할 해당 예외를 catch문에 추가
    2. 모든 Exception 클래스의 최상위 부모인 Exception 타입의 catch문 맨 마지막에 추가 (catch(Exception e)) —> 항상 catch문 마지막에 위치해야 한다

Finally

  • try-catch 문 마지막에 finally {} 추가하면 예외가 발생 하던 안하던 무조건 실행

throws

  • throws를 이용하여 특정 메소드에서 발생된 예외를 메소드를 호출한 쪽으로 던질 수 있다.

  • 최악 실행 시간을 사용 -> Big O

Hash Table

  • Hash 란 임의 값을 고정 길이로 변환한것
  • Key에 데이터(value)를 매핑할 수 있는 데이터 구조
  • 해쉬 함수를 통해 배열에 키에 대한 데이터를 저장할 수 있는 주소(index)를 계산
  • key를 통해 바로 데이터가 저장된 주소(index)를 알 수 있어 저장 및 탐색 속도가 획기적으로 빠르다
  • 미리 해쉬 함수가 생성할 수 있는 주소에 대한 공간을 배열로 할당, 키에 따른 데이터 저장 및 탐색을 지원 (메모리 공간과 탐색 시간을 맞바꾸는 기법으로도 불림)
  • Python에서는 별도 구현 X, 딕셔너리 타입 사용

동작 방식

  1. Key(1)를 Hash fucntion(2)에 입력하면 주소(index number) 출력
  2. 출력된 주소(index number)를 (3)에서 찾아 필요한 데이터(4) 찾을 수 있다.
장점
  • 데이터 R / W 속도가 빠르다
  • 해쉬는 키에 대한 데이터가 있는지 (중복) 확인이 쉽다
단점
  • 일반적으로 저장공간이 기존보다 더 많이 필요하다
  • 여러 키에 해당하는 주소가 동일할 경우 충돌을 해결하기 위한 별도의 자료구조가 필요하다
사용
  • 검색이 많이 필요할 때
  • 저장, 삭제, 읽기가 자주 쓰일 때
  • 캐쉬 구현시 (중복 확인이 쉬워서)

Queue

  • 줄을 서는 행위같은 개념을 추상화한 데이터 구조
  • 가장 먼저 넣은 데이터를 제일 먼저 처리하는 구조 (FIFO)
  • 사용되는 메소드
    • Enqueue : 큐에 데이터 입력
    • Dequeue : 큐에 데이터 제거
  • Python 에서 Queue 기능을 제공하는 메소드를 사용하여 큐 구현 가능 (import queue)

Stack

  • 마지막에 들어온 데이터를 제일 먼저 처리하는 추상적인 데이터 구조
  • 사용되는 메소드
    • push : 스택에 데이터 입력
    • pop : 스택에 데이터 제거
  • 장점
    • 구조가 단순해서 구현이 쉽고 데이터 R/W 속도가 빠르다
  • 단점
    • 데이터 갯수를 미리 정해야 해서 메모리 낭비가 발생할 수 있다.
  • Python 에서 리스트 기능을 제공하는 메소드를 사용하여 스택을 구현할 수 있다. (append(push), pop) 메소드 제공

Linked List

  • 원소와 다음 원소의 메모리 주소를 가르키는 포인터로 이루어진 데이터 구조
  • 처음을 의미하는 head, 마지막을 의미하는 tail이 있을수 있다.
  • 장점
    • 미리 메모리 공간을 할당하지 않아도 됨
  • 단점
    • 원소 외에 포인터를 가지기 때문에 저장공간 효율이 높지 않다
    • 특정 정보를 찾을 때 속도가 느리다
    • 중간 데이터를 삭제하면 앞 뒤 연결을 재구성해야 하는 부가적 작업 필요
  • 사용자 수가 20만명으로 예상되는 웹 페이지의 회원 정보를 배열로 관리할 때 25만개의 배열을 미리 할당해야 하는데 사용자가 10만명밖에 되지 않거나 25만명을 초과할 때 문제가 생길 수 있다, 하지만 Linked List를 사용하면 회원 한명이 가입할 때마다 노드를 하나씩 생성하므로 메모리를 정확하게 사용할 수 있다.

인스턴스 생성과 힙 메모리

인스턴스 (instance)
  • 클래스는 객체의 속성을 정의, 기능을 구현하여 만들어 놓은 코드 상태
  • 실제 클래스 기반으로 생성된 객체(인스턴스)는 각각 다른 멤버 변수 값 가짐
  • new 키워드를 사용하여 인스턴스 생성
힙 메모리 (heap memory)
  • 생성된 인스턴스는 동적 메모리 (heap memory) 에 할당됨
  • C나 C++ 언어에서는 사용한 동적 메모리를 프로그래머가 해제 시켜야 하지만 자바에서는 Gabage Collector가 주기적으로 사용하지 않는 메모리 해제
  • 하나의 클래스로 부터 여러개의 인스턴스가 생성되고 각각 다른 메모리 주소를 가지게 됨

생성자 (constructor)

생성자
  • 오직 객체를 생성하기 위해 new와 함께 호출 className instanceName = new className(); 에서 className()이 생성자
  • 객체가 생성될 때 변수나 상수를 초기화 하거나 다른 초기화 기능을 수행하는 메소드 호출
  • 반환값이 없고 클래스 이름과 동일
  • 대부분 생성자는 외부 접근이 가능하지만 필요하다면 private으로 선언할수도 있다.
  • this 는 변수나 메소드 앞에 사용되며 this 사용시 매개변수가 아닌 멤버 변수를 의미
  • 맴버변수들의 값을 모두 default로 초기화
기본 생성자 (default constructor)
  • 클래스에 반드시 하나이상 존재
  • 생성자를 구현하지 않아도 new 키워드와 함께 생성자를 호출 가능
  • 클래스 생성자 하나도 없는 경우 컴파일러가 생성자 코드 넣어준다 (public className(){})
  • 매개변수, 구현부가 없다
접근 제어자
  • Public
    • 다른 패키지에서 사용 가능
    • 다른 어떤 클래스에서도 public class는 사용 가능
  • Private
    • private이 붙은 변수나 메소드는 해당 클래스에서만 접근이 가능
    • 다른 클래스에서 접근이 불가능
  • Default
    • 접근 제어자를 별도로 설정하지 않는다면 해당 패키지 내에서만 접근이 가능
    • **동일 패키지**, 다른 클래스에서 접근 가능
  • Protected
    • 이 제어자가 붙은 변수, 메소드는 동일 패키지의 클래스 또는 해당 클래스를 상속받은 다른 패키지의 클래스에서만 접근이 가능 (상속 → import packageName.ClassName;)
패키지 (package)
  • 소문자로만 구성
  • 여러 디렉토리로 구성되면 . 을 이용해 구분 (package school. high)
  • 주석을 제외하고 클래스 파일의 가장 첫 줄에 위치
  • import packageName.ClassName; 다른 패키지의 클래스 사용하려면 있어야됨
